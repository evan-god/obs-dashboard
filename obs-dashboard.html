📥 OBS 日志服务 — 部署说明

请先确认你是哪种情况：

🖥 场景 A：我要一台新电脑当监控大盘

电脑是全新的，或者刚重装系统，
想把这台电脑变成「总控台」来监控所有直播间。

🎥 场景 B：我要给直播间 OBS 电脑装日志服务

直播间电脑已经在运行 OBS 直播了，
想让监控大盘能远程查看它的日志。

🖥 场景 A：全新安装监控大盘（新电脑当总控台）

监控大盘软件是绿色免安装的，不需要安装任何东西，直接复制文件夹就能用。

📂 第一步：复制监控大盘文件夹

  1. 在当前这台监控电脑上，找到 exe 所在的文件夹
    （里面有 OBS直播间监控大盘.exe、obs-dashboard.html 等文件）
  2. 把整个文件夹复制到 U 盘（或通过网络共享发送）
  3. 在新电脑上，把文件夹粘贴到任意位置（建议放 D 盘根目录，如 D:\obs-dashboard）

🚀 第二步：双击运行

  1. 打开文件夹，找到 OBS直播间监控大盘.exe
  2. 双击运行
  3. 软件会自动打开窗口，显示监控大盘界面

🟢 第三步：确认连接

  1. 监控大盘打开后，左侧会显示已配置的直播间列表
  2. 点击某个直播间，确认能看到推流画面和控制面板
  3. 如果看不到直播间，需要先在 HTML 里配置直播间 IP 地址（见下方说明）

✅ 新监控电脑 不需要安装的东西：

❌ 不需要装 Node.js  — Electron 自带运行环境
❌ 不需要装 OBS     — 除非这台电脑也要直播
❌ 不需要装任何依赖  — 所有文件都在文件夹里
❌ 不需要管理员权限  — 直接双击就能运行

监控电脑唯一需要的条件：
✅ 能和 OBS 直播间电脑互相 ping 通（同一局域网或 VPN）
✅ 防火墙允许本机访问远程 4455 和 8393 端口（出站默认放行，一般不用管）

💡 如果想看监控电脑自己的 OBS 日志（这台电脑也在跑 OBS 的话），
需要给它也部署日志服务 → 看下面的「场景 B」。

🎥 场景 B：给 OBS 直播间主机部署日志服务

下面要做的，是给每一台运行 OBS 的直播间电脑装一个小程序。
装好之后，你在这台监控大盘上点一下按钮，就能直接看到那台电脑的 OBS 日志。

第一步：准备工作（在监控大盘这台电脑上做）

📍 找到这两个文件：

打开 obs-dashboard.exe 所在的文件夹，你会看到同目录下有：

📄 obs-log-service.js（日志服务主程序）
📄 install-obs-log-service.bat（一键安装脚本，双击即可）

💡 把这两个文件复制到 U 盘，或者通过微信/QQ 发送给自己，后面要放到 OBS 直播间电脑上。

第二步：安装 Node.js（在 OBS 直播间电脑上做）

Node.js 是什么？ 它是一个运行环境，就像你要运行 Python 脚本需要先装 Python 一样，
要运行 .js 文件就需要 Node.js。装一次就行，以后不用管。

📥 下载安装（约 2 分钟）：

  1. 打开浏览器，访问 https://nodejs.org/
  2. 点击左边的绿色大按钮（写着 LTS），会自动下载安装包
  3. 下载完成后双击运行，一路点 "Next"（下一步），全部用默认选项
  4. 最后点 "Install"（安装），等待进度条走完
  5. 完成后点 "Finish"（完成）

🔰 别担心，Node.js 是免费开源的，全球几百万开发者都在用，安全无毒，装完不会影响电脑其他功能。

第三步：部署日志服务（在 OBS 直播间电脑上做）

📂 复制文件到这台电脑：

把第一步准备好的两个文件（obs-log-service.js 和 install-obs-log-service.bat）
放到这台电脑的桌面上。放在哪里都行，桌面最方便。

🚀 双击运行安装脚本：

  1. 在桌面上找到 install-obs-log-service.bat（齿轮图标）
  2. 双击打开它
  3. 会弹出一个黑色窗口，自动执行以下操作（不需要你动手）：
[1/5] 检测 Node.js ...................... ✓ 找到
[2/5] 配置防火墙（开放端口 8393）....... ✓ 已添加
[3/5] 安装服务文件到 C:\Users\...\obs-log-service\ ... ✓ 完成
[4/5] 创建开机自启 ..................... ✓ 已创建
[5/5] 启动服务 ......................... ✓ 服务已启动

看到「✓ 部署完成！」就说明成功了。如果提示「未找到 Node.js」，说明第二步没装好，
回到第二步下载安装 Node.js，安装完后再双击一次 .bat。

📍 服务安装在哪里？→ 自动安装到 C:\Users\你的用户名\AppData\Roaming\obs-log-service\
🔄 开机会自动启动吗？→ 会，已添加到 Windows 启动文件夹
🗑 桌面上的两个文件可以删吗？→ 可以，安装完成后就可以删掉

第四步：验证部署是否成功（可选，确认一切正常）

在 OBS 直播间电脑上，按键盘 Win + R，输入 cmd 回车，打开黑色命令行窗口，输入：

curl http://127.0.0.1:8393/status

如果看到 {"ok":true,"host":"你的电脑名",...} 就说明一切正常！

💡 这一步不是必须的。你也可以直接回到监控大盘，点一下对应主机的 OBS 日志按钮，能加载出来就说明部署成功了。

📋 完整清单：OBS 直播间电脑需要具备什么？

① OBS 直播软件 已安装并至少运行过一次（这样才会有日志文件）
② OBS WebSocket 在 OBS → 工具 → WebSocket 服务器设置 → 开启端口 4455
③ Node.js 已安装（在这台 OBS 电脑上，不是监控大盘电脑）
👉 https://nodejs.org/ 下载 LTS 版本
④ 防火墙 端口 4455 和 8393 允许入站（安装脚本自动处理 8393）
⑤ 网络 与监控大盘在同一局域网（同一个路由器下），或通过 VPN 互通

❓ 常见问题：

Q: 这个服务会占用很多电脑资源吗？
A: 不会。它只是一个极轻量的小程序，平时几乎不占 CPU 和内存，完全不影响 OBS 直播。

Q: 会影响 OBS 直播推流吗？
A: 完全不会。日志服务只读日志文件，跟 OBS 推流没有关系。

Q: 万一 OBS 日志路径不是默认的怎么办？
A: 程序会自动查找常见的 OBS 日志位置（标准安装、便携版等），通常不需要手动配置。

Q: 监控大盘显示"无法连接"怎么办？
A: 检查：① OBS 电脑是否开机 ② Node.js 是否已安装 ③ 防火墙是否开放了 8393 端口
    ④ 重新双击一次 .bat 确认服务在运行

Q: 如何卸载？
A: 删除 C:\Users\你的用户名\AppData\Roaming\obs-log-service\ 文件夹，
    然后从启动文件夹删除 obs-log-service.vbs 快捷方式即可。

🎉 全部完成后，回到监控大盘点击对应主机的「OBS 日志」按钮，就能看到日志文件列表（含名称/大小/时间），选择任意文件查看。
